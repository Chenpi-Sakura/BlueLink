## 1. 产品概述
### 1.1 产品愿景
蓝链（BlueLink.）是一款面向深度学习者与内容创作者的端侧AI认知溯源引擎。它的目标是实现AI从“答案生成器”到知识的索引与溯源工具的转变，强迫并引导用户回归原始语境，进行主动思考与知识内化。产品通过“溯源降噪”过滤重复信息，运用“引力线”将灵感直连原文，构建个性化知识图谱，最终让用户以最短路径抵达真相源头。
### 1.2 核心价值主张
+ **AI不生产答案，只提供索引与溯源**：打破“虚假获得感”，强迫用户回归原文阅读。
+ **知识不出端，隐私即自由**：敏感数据本地处理，构建绝对安全的个人“第二大脑”。
+ **增量降噪，只给干货**：自动识别并折叠重复内容，仅展现真正的“信息增量”。
+ **费曼伴学，自我校准**：通过比对用户表述与原文逻辑，定位认知偏差，引导主动内化。
+ **个性化溯源粒度**：用户可自由调控提示明确度、探索深度与溯源颗粒度。
### 1.3 目标场景
+ 高校科研工作者进行文献研读、综述撰写、理论溯源。
+ 内容创作者深度阅读材料，提炼观点并记录灵感。
+ 终身学习者构建个人知识体系，抵御信息过载与AI依赖。

### 1.4 产品差异化
| 对比维度 | 蓝链 BlueLink    | 竞品（Obsidian/Flomo/闪念贝壳） |
| ---- | -------------- | ----------------------- |
| AI角色 | 索引与溯源，不直接给答案   | 生成、总结或被动存储              |
| 知识处理 | 实时去重、增量折叠、原文锚点 | 手动链接、简单标签或仅速记           |
| 隐私安全 | 敏感数据端侧本地处理     | 多数依赖云端                  |
| 交互范式 | 锚点穿梭、引力线、聚光灯阅读 | 传统气泡或单一列表               |
| 学习理论 | 内置费曼学习法偏差提示    | 无内置学习校验                 |

## 2. 用户分析
### 2.1 目标用户画像
+ **核心受众**：具有中高强度深度学习需求的群体，如高校科研人员、硕士/博士生、内容创作者、知识型博主。
+ **年龄区间**：20-40岁。
+ **行为特征**：
    - 长期进行资料研读、论文阅读与撰写。
    - 需要频繁记录并整理灵感碎片。
    - 担心AI导致思维惰性，希望保持独立思考能力。
    - 对数据隐私较为敏感，不愿将未成形想法或敏感文档上传云端。
### 2.2 核心痛点
1. **思维惰性与虚假获得感**：依赖AI总结，误以为已掌握知识，削弱独立推演能力。
2. **信息重复噪音**：海量文档中大量背景、常识性内容重复，消耗筛选精力。
3. **隐私焦虑**：云端AI服务无法保证个人灵感与敏感资料的绝对安全，导致用户不敢真实表达。
4. **知识孤岛**：不同来源的笔记、灵感与文献彼此割裂，难以发现深层关联。

## 3. 产品架构与核心逻辑
蓝链整体分为四大核心模块，覆盖从知识摄入、处理、内化到关联的全流程：

```plain
[灵感/文档导入] → [知识库（去重、切片）] → [对话溯源（锚点/费曼伴学）]
                         ↕
[阅读器（聚光灯/折叠）] ← → [知识图谱（引力线连接）]
```

**核心闭环流程**：捕获 → 入库降噪 → 溯源提问 → 锚点跳转原文 → 聚光灯阅读 → 灵感笔记关联 → 图谱生长。
**AI协作模式**：端侧大模型处理个人敏感数据与常规推理；云端大模型承担高计算量的文档向量化、全库比对等任务。敏感数据划分由用户手动标记及AI自动识别共同完成。

## 4. 功能需求详述
### 4.1 知识库与文档管理
**模块定位**：用户所有文献、灵感、多模态数据的统一管理中心，具备智能去重与提示能力。

| 功能点              | 详细描述                                                              | 优先级 |
| ---------------- | ----------------------------------------------------------------- | --- |
| 多格式导入            | 支持PDF、Word、Markdown、HTML、纯文本、图片OCR、语音转文字、网页剪藏。                    | P0  |
| 入库去重与增量折叠        | 导入新文档时，AI比对已有知识库，自动识别重复概念与已知背景，将其折叠。只向用户高亮展示“信息增量”内容（如新观点、反常识数据）。 | P0  |
| 概念路标（代替原有的自动化摘要） | 提取文章试图回答的核心问题或争议点（例如：“本文探讨了XX与YY的关系，并对传统ZZ理论提出了质疑”）               | P0  |
| 原文切片与索引          | 文档解析后按段落、句子语义切片存储，建立向量索引，支持精准定位任何概念所在原文片段。                        | P0  |
| 灵感快速捕获           | 首页悬浮快捷栏支持文字速记、语音录入、拍照/扫描多模态输入，一键生成灵感卡片并入知识库。                      | P0  |
| 手动标记与分类          | 用户可自定义标签、文件夹，标记文档为“私密/敏感”，敏感文档即锁定端侧处理。                            | P1  |
| 版本与来源追溯          | 保留文档原始来源与修改记录，每条灵感与原文的关系可回看。                                      | P1  |

**交互细节**：文库首页采用瀑布流卡片布局，每张卡片显示标题、摘要、来源、时间戳。卡片附带轻微阴影与圆角，长按可呼出更多操作。

### 4.2 对话与溯源引擎
**模块定位**：用户进行概念提问、费曼学习的核心界面，强调“只给路径，不给答案”。

| 功能点 | 详细描述 | 优先级 |
| --- | --- | --- |
| 概念提问与锚点生成 | 用户输入疑问，AI在知识库中检索，返回一段引导引言及若干“锚点卡片”，卡片内容为原文切片的关键句及定位信息。 | P0 |
| 锚点跳转与聚光灯联动 | 点击锚点卡片，自动跳转至文档内对应段落，触发聚光灯模式（见4.3阅读器）。 | P0 |
| 提问历史与线索保留 | 每一次提问生成的引言和锚点集合可作为线索卡片保留在对话流中，支持回看。 | P1 |
| 溯源粒度动态调节 | 全局或临时切换：文章级粒度（溯源至篇章/小节）、句词级粒度（定位到具体句子或术语）。 | P0 |
| 提示明确度调节 | 设置档位：直接指出逻辑漏洞并给出原文对比；或仅抛出矛盾的原文段落，由用户自行发现错误。 | P1 |
| 探索深度控制 | 标准：仅回答当前疑问；深度探索：AI在提供溯源后主动推荐关联度高的争议性文献或对立的观点。 | P1 |
| 对话中断与恢复 | 支持随时跳出阅读，返回后对话状态保留。 | P2 |

**交互规则**：对话界面弱化传统气泡，信息呈现在卡片中。AI的响应不使用长文本，仅提供引言（最多三句话）和锚点列表。输入框底部常驻“溯源粒度”快捷切换开关。

### 4.3 智能阅读器
**模块定位**：提供两类阅读模式，分别服务于溯源跳转与自主浏览，核心在于“降噪”与“聚焦”。
#### 4.3.1 聚光灯模式（由锚点触发）
+ **触发条件**：从对话锚点跳转进入。
+ **表现**：目标段落维持正常对比度（黑字白底），其余上下文降低透明度至30%并呈现灰态，但文字仍可阅读（不缺失）。
+ **退出**：点击屏幕任意空白处，聚光灯消散，平滑过渡为全文正常显示。
+ **附加**：在目标段落旁显示一条垂直的“引力线”指示器，标注跳转来源。
#### 4.3.2 增量折叠模式（由文库主动打开）
+ **触发条件**：从文库卡片直接打开文档。
+ **表现**：文档完整保留，但AI识别出的重复背景、已知概念段落被“内联折叠”，显示为一条极简折叠提示条：“AI已折叠 XXX 字重复背景，点击展开”。
+ **高亮**：未被折叠的“信息增量”段落边缘可有淡蓝色左侧边框标记。
+ **交互**：点击折叠条可展开，再次点击收起。
#### 4.3.3 通用阅读功能：
+ 支持目录导航、全文搜索。
+ 阅读进度保存与书签。
+ 支持从阅读界面快速摘录并生成灵感卡片（长按选中文字 → 发送至灵感库）。

### 4.4 知识图谱
**模块定位**：将用户文档、灵感碎片、概念实体编织成一张动态星图，揭示隐性关联。

| 功能点 | 详细描述 | 优先级 |
| --- | --- | --- |
| 节点自动生成 | AI从文档与灵感中抽取实体、概念作为节点，灵感和文档本身也为节点。 | P0 |
| 引力线连接 | AI自动发现节点间的相似、引用、矛盾、补充等关系，绘制连线并标注关系类型（如“支撑”、“质疑”）。 | P0 |
| 无极缩放与局部聚焦 | 图谱支持手势缩放；点击节点，镜头拉近并居中，高亮该节点及一阶强关联节点，其余淡化为背景。 | P0 |
| 节点类型区分 | 灵感节点、文献节点、概念节点采用不同形状或颜色标识。 | P1 |
| 手动增删连线 | 用户可拖拽建立新关系或删除错误关系，AI据此优化模型。 | P1 |
| 溯源路径回放 | 选中两个节点，可展示它们之间的最短关联路径，突出关键节点。 | P2 |

### 4.5 费曼伴学系统
**模块定位**：基于费曼学习法的主动知识校验工具，迫使学习者用自己的话解释概念，AI仅负责匹配原文并指出偏差。

| 功能点        | 详细描述                                                        | 优先级 |
| ---------- | ----------------------------------------------------------- | --- |
| 费曼挑战启动     | 用户在对话界面选择“费曼模式”，AI提示用户就某概念进行用自己的语言复述解释。                     | P0  |
| 实时语音/文字输入  | 支持语音输入（端侧转文字）或键盘输入解释内容。                                     | P0  |
| 偏差识别与引力线绑定 | AI在本地知识库中比对用户表述与原文，识别逻辑漏洞、事实错误或遗漏关键点；将错误部分与对应原文片段建立“引力线”连接。 | P0  |
| 偏差报告生成     | 生成通俗的纠偏报告，不直接显示正确表述，而是高亮偏差处并附原文定位锚点。                        | P0  |
| 多轮追踪       | 用户根据锚点阅读原文后，可再次解释，系统持续校准并记录理解进步。                            | P1  |

### 4.6 灵感捕获与聚合
**模块定位**：记录一切碎片想法，后台自动进行语义关联与图谱集成。
+ **快捷入口**：首页底部悬浮快捷栏，三个按钮——文字速记、语音、拍照/扫描。
+ **灵感卡片**：每条灵感生成独立卡片，AI自动提取关键词并打标签。卡片随文库瀑布流展示。
+ **后台关联**：灵感保存后，系统异步分析其内容，自动搜寻已有知识库及旧想法中的关联点，生成引力线，并在图谱中呈现。
+ **灵感演变**：允许将多张卡片合并、关联，逐步形成文章大纲或论文框架。
### 4.7 隐私与端侧处理
+ **敏感数据识别**：用户可手动将文档/灵感标记为“私密”；AI也可根据内容自动建议标记。
+ **端侧模型覆盖**：敏感数据的切块、检索、费曼比对、灵感关联等全部在端侧完成，数据不出设备。
+ **云端协同边界**：非敏感大篇幅文档的向量化、全局去重比对可调用云端算力，但原始文本可根据用户设置决定是否上传（默认不上传，仅上传脱敏的向量特征）。
+ **本地加密存储**：知识库整体加密存储在设备本地，并可同步到用户私有云（iCloud/WebDAV等），传输过程加密。
+ **权限与隐私声明**：首次使用明确说明数据处理方式，提供详细隐私设置。
### 4.8 认知分层与个性化
+ **用户画像构建**：初次使用引导填写职业、专业领域、兴趣、价值观关键词。系统也在使用中动态学习用户的知识背景与常用概念层级。
+ **语境适配**：AI在生成引言、线索时，优先采用用户画像中的术语体系。例如对物理系学生使用“哈密顿量”而非“能量函数”。
+ **伴读风格切换**：提供“温和启发”与“直指核心”等多种语气，或根据当前任务紧急度自动调整。
+ **知识盲区感知**：当提问中涉及用户尚未入库的概念时，AI可提示“这一概念暂未在你的知识库中找到，是否联网搜索或手动添加？”（联网功能可配置开关）。

## 5. 交互设计规范
### 5.1 核心交互原则
1. **拒绝投喂，只给路径**：任何AI响应都不得直接给出最终答案，必须通过锚点或引力线引导用户返回原文。
2. **一触即达**：缩短从产生疑问到查看原文的步长，点击锚点立即跳转。
3. **空间方位感**：通过磨砂玻璃、渐进式界面，让用户始终清楚自己所处位置。
4. **动态调控权**：溯源粒度、提示明确度等控制项始终触手可及。
### 5.2 锚点穿梭机制详解
+ 在对话界面，AI返回的每个锚点卡片包含：文档标题、来源片段前20字、跳转按钮。
+ 点击锚点，文档阅读器以模态或全屏方式展开，并自动滚动至目标段落，聚光灯亮起。
+ 用户验证后，可通过左滑手势或关闭按钮返回对话，对话历史保留该锚点已读状态。
### 5.3 手势与快捷操作
+ 双击阅读页面空白区 → 退出聚光灯/展开所有折叠。
+ 长按锚点卡片 → 预览原文小窗（Peek）。
+ 在灵感输入框下滑 → 切换到语音输入。
+ 图谱中双指缩放，单指拖动，双击节点聚焦。
### 5.4 界面流转逻辑
（流程图示意，在文档中可文字描述关键路径）
+ **提问 → 溯源 → 阅读 → 返回**：
    1. 用户在对话视图输入问题。
    2. AI生成引言与锚点卡片列表。
    3. 用户点击某锚点，界面从对话平滑过渡到阅读器（聚光灯模式）。
    4. 用户点击空白处或返回按钮，阅读器滑出，对话界面重现。
+ **灵感捕获 → 卡片生成 → 图谱更新**：
    1. 用户在任何界面呼出快捷栏，录入灵感。
    2. 卡片自动追加到文库瀑布流顶部。
    3. 后台处理完成后，图谱中出现新节点，并动态生长引力线，可提示用户查看。

## 6. 视觉设计规范
### 6.1 设计哲学
“人文底色，极简透视”。平衡古典纸本阅读的静谧与现代AI工具的锐利，让机器隐匿于文字之后。

### 6.2 色彩体系
+ **背景主色**：米白色 (#FAF7F2) / 羊皮纸暖色，模拟纸质阅读感。
+ **强调色**：克莱因蓝 (#002FA7)，用于所有超链接、按钮、引力线、高亮边线、锚点卡片边框。
+ **文字色**：主文本 (#2C2C2C)，次要文本 (#6B6B6B)。
+ **折叠/灰态**：灰色 (#B0B0B0) 透明度30%应用于聚光灯下的非焦点文本。
+ **磨砂玻璃**：白色半透明配合高斯模糊值约20px，用于悬浮栏、半屏卡片背景。
### 6.3 字体规范
+ **阅读/内容区域**：采用古典衬线体（如中文使用宋体/明体衍生，西文使用 Garamond 或 Caslon），字号基准16pt，行间距1.8倍。
+ **UI控件/标题**：使用无衬线体（如中文苹方/思源黑体，西文SF Pro Display或Inter），保持现代感。
### 6.4 形状与材质
+ 卡片：大圆角（16px），轻量弥散阴影（Y偏移2px，模糊半径8px，透明度10%）。
+ 按钮与输入框：圆角24px。
+ 悬浮元素：高斯模糊背景，边缘半透。
+ 图标：线性图标，描边2px，颜色统一采用中性灰或克莱因蓝。
### 6.5 布局原则
+ 大量留白，界面元素间距宽松，减少视觉压迫。
+ 核心界面对话、阅读、文库、图谱各自保持独立空间，通过底部导航栏切换。
+ 底部导航图标+文字，当前选中项使用克莱因蓝强调。

## 7. 技术实现要求
### 7.1 AI模型协同架构
+ **端侧大模型**：
    - 任务：敏感数据语义理解、切片检索、费曼偏差比对、小型关系抽取、个性化语境生成。
    - 要求：可运行于主流移动芯片（如Apple Neural Engine、高通Hexagon），支持量化，内存占用低于1.5GB。
    - 推荐使用LLaMA 3 8B量化版或专用SLM，借助MLX/CoreML/MediaPipe等框架部署。
+ **云端大模型**：
    - 任务：大文档向量化、跨文档全量去重比对、复杂知识图谱全局关系发现、高计算量摘要生成。
    - 安全策略：文档上传前脱敏，仅向量和元数据入云；或由用户授权。

### 7.2 数据存储与向量数据库
+ 端侧向量数据库（如Chroma Mobile或SQLite+向量扩展）用于本地切片存储与检索。
+ 文档原文与元数据使用加密本地数据库。
+ 云端可同步使用Qdrant或Milvus进行大规模检索（仅非敏感数据）。

### 7.3 关键算法与实现
+ **文档解析与切片**：支持多种格式解析，按照段落、语义边界进行切片，保持上下文完整。
+ **去重与增量检测**：利用文本相似度（如SimHash结合语义向量余弦相似度）比对新老内容，标记重复段落。
+ **实体抽取与图谱构建**：基于端侧NER模型抽取概念、作者、引文等，结合共现关系与语义推理建立连线。
+ **费曼偏差检测**：将用户解释文本与原文切片编码为向量，利用自然语言推理（NLI）模型判断矛盾、中立或蕴含关系，定位不匹配片段。

### 7.4 端云数据流控制
+ 用户标记为“私密”的文档，其全文和向量绝对停留在本地。
+ 非私密文档的向量可上传云端增强检索，原文根据用户设置决定（默认不上传）。
+ 同步策略：支持iCloud或WebDAV端到端加密同步知识库和设置。

## 8. 非功能需求
### 8.1 性能
+ 锚点跳转文档定位至段落，响应时间 < 500ms。
+ 去重比对（本地）对于1MB文档应在3秒内完成。
+ 知识图谱加载并渲染1000个节点无明显卡顿（60fps）。

### 8.2 安全与隐私
+ 本地数据库全量加密（256-bit AES）。
+ 传输过程使用TLS 1.3。
+ 严格遵守《个人信息保护法》，提供数据导出与删除功能。

### 8.3 可用性
+ 首次使用引导不超过3屏，关键功能有直观教程。
+ 核心任务（提问-溯源-阅读）完成步数不超过3步。
+ 支持无障碍化，包括字体缩放、VoiceOver适配。

### 8.4 兼容性
+ iOS 16+，Android 13+。
+ 适配不同屏幕尺寸（手机为主，可延展至平板）。

### 8.5 可扩展性

## 9. 附录
### 9.1 术语表
+ **锚点（Anchor）**：AI从原文提取的定位线索，可点击跳转到具体段落。
+ **引力线**：代表概念、文档、灵感之间逻辑关系的可视化连线。
+ **聚光灯模式**：阅读器中高亮目标段落、弱化其余内容的显示状态。
+ **增量折叠**：将用户已熟悉的重复内容自动收起，仅显示新信息。
+ **溯源颗粒度**：AI返回线索的定位精度级别，如文章级或句词级。
+ **费曼伴学**：引导用户用自己的语言解释概念并与原文比对的学习模式。

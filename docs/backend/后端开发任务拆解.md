## 架构决策（2026-06-21 修订）

### 后端存储：不加密明文存储

| 维度 | 原设计（V2.1） | 现决策 | 理由 |
|------|---------------|--------|------|
| 切片文本存储 | Fernet 加密 → `LargeBinary` | **纯文本 `TEXT`** | 比赛演示场景，无敏感数据 |
| 传输加密 | HTTPS | HTTPS | 保持不变 |

**影响范围：**
- `segments.text_encrypted`（BLOB）→ `segments.text`（TEXT），**不再加密**
- 服务端不管理任何加密密钥，`security.py` 中的 Fernet 加解密不再使用
- `requirements.txt` 中 `cryptography` 依赖可移除
- 手机端 `SecurePrefs` 独立运作，不受影响

### 数据库：SQLite → PostgreSQL + pgvector

| 维度 | V2.1 原设计 | 现决策 | 理由 |
|------|------------|--------|------|
| 数据库 | SQLite | **PostgreSQL 17** | 服务端优先，多设备支持 |
| 向量存储 | numpy npy 文件 | **pgvector**（PostgreSQL 扩展） | 统一存储，支持 SQL 级向量检索 |
| 连接管理 | 单连接 | **连接池**（pool_size=10） | 生产级并发 |

### 部署：Docker 容器化

```
docker-compose.yml
├── service: db     → pgvector/pg17（PostgreSQL + 向量扩展）
└── service: app    → FastAPI（wait db healthy 后启动）
```

**启动命令：**
```bash
docker compose up -d                    # 一键启动
docker compose logs -f                  # 看日志
docker compose down -v                  # 停止并清数据
```

- `vectors/store.py` 是共享瓶颈，**必须先做**

- 费曼 / 图谱 / 卡片+同步 三条链路**互相独立**，可并行开发

## 八、踩坑记录

> 开发过程中遇到的问题与解决方案，供后续参考。

### 1. Dockerfile 构建顺序错误

**问题：** `RUN pip install -r requirements.txt` 在 `COPY . .` 之前执行，但 `requirements.txt` 还没复制进去，导致找不到文件。

**解决：** 先 `COPY requirements.txt .` 再 pip install，最后 `COPY . .`。这个顺序也利用了 Docker 层缓存——requirements.txt 不变时跳过 pip 安装。

### 2. libgl1-mesa-glx 在 Debian trixie 不存在

**问题：** `python:3.12-slim` 基于 Debian trixie，`libgl1-mesa-glx` 已不再提供此包。

**解决：** 后端只做 PDF 文本提取（PyMuPDF），不需要 OpenGL 渲染，直接移除该依赖。

### 3. .env 被提交到 git 仓库

**问题：** `.gitignore` 没有 `.env` 规则，`git add -A` 把含 API Key 的 `.env` 提交了。API Key 暴露在 git 历史中。

**解决：** 
- 添加 `.env` 到 `.gitignore`
- `git rm --cached .env` 停止跟踪
- **去 DeepSeek 后台撤销旧 key 并重新生成**

### 4. pip 镜像源在国内服务器下载慢

**问题：** 从 pypi.org 下载 PyMuPDF（25MB）和 scipy（35MB）速度极慢。

**解决：** Dockerfile 默认使用阿里云镜像 `mirrors.aliyun.com/pypi/simple/`，apt 使用清华镜像 `mirrors.tuna.tsinghua.edu.cn`。

### 5. tests/ 被 .dockerignore 排除导致测试没法跑

**问题：** `.dockerignore` 里有 `tests/`，Docker 镜像里没有测试文件，`docker exec pytest` 找不到测试。

**解决：** 移除 `.dockerignore` 中的 `tests/` 行。

### 6. pgvector Vector(1024) 类型不兼容 SQLite

**问题：** `SegmentVector` 模型使用 `Vector(1024)` 列类型（pgvector 专用），SQLite 无法创建该表。导致 `conftest.py` 使用内存 SQLite 时所有表的 `create_all` 失败。

**解决：** 彻底移除 SQLite，全栈使用 PostgreSQL。`conftest.py` 改为使用配置的 PostgreSQL 数据库 + 事务隔离。

### 7. Android 与后端字段命名不一致（camelCase vs snake_case）

**问题：** Android 端 DTO 使用 camelCase（`userExplanation`），后端 Pydantic 默认 snake_case（`user_explanation`），导致请求解析和响应序列化 mismatch。

**解决：** 在 Pydantic model 上设置 `alias_generator=to_camel` + `populate_by_name=True`，在路由上使用 `response_model_by_alias=True`。`from` 字段（Python 关键字）用 `alias="from"` 单独处理。

### 8. pytest-mock 包未预装

**问题：** 测试中使用 `mocker` fixture，但 `pytest-mock` 不在 `requirements.txt` 中。

**解决：** 改用内置的 `monkeypatch` fixture，无需额外依赖。

### 9. 费曼 API 缺少 X-User-Id 时状态码是 422 不是 400

**问题：** 测试预期 `get_user_id` 依赖返回 400（HTTPException），但 FastAPI 的 Header 校验在依赖执行前就返回了 422。

**解决：** 测试预期改为 422。这是 FastAPI 的行为：Pydantic 模型校验返回 422，路由内 HTTPException 才返回 400。

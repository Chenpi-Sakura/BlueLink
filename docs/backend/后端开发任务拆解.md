## 架构决策（2026-06-21 修订）

### 后端存储：不加密明文存储

| 维度 | 原设计（V2.1） | 现决策 | 理由 |
|------|---------------|--------|------|
| 切片文本存储 | Fernet 加密 → `LargeBinary` | **纯文本 `TEXT`** | 比赛演示场景，无敏感数据 |
| 传输加密 | HTTPS | HTTPS | 保持不变 |

**影响范围：**
- `segments.text_encrypted`（BLOB）→ `segments.text`（TEXT），**不再加密**
- 服务端不管理任何加密密钥，`security.py` 中的 Fernet 加解密不再使用
- `requirements.txt` 中 `cryptography` 依赖可移除
- 手机端 `SecurePrefs` 独立运作，不受影响

### 数据库：SQLite → PostgreSQL + pgvector

| 维度 | V2.1 原设计 | 现决策 | 理由 |
|------|------------|--------|------|
| 数据库 | SQLite | **PostgreSQL 17** | 服务端优先，多设备支持 |
| 向量存储 | numpy npy 文件 | **pgvector**（PostgreSQL 扩展） | 统一存储，支持 SQL 级向量检索 |
| 连接管理 | 单连接 | **连接池**（pool_size=10） | 生产级并发 |

### 部署：Docker 容器化

```
docker-compose.yml
├── service: db     → pgvector/pg17（PostgreSQL + 向量扩展）
└── service: app    → FastAPI（wait db healthy 后启动）
```

**启动命令：**
```bash
docker compose up -d                    # 一键启动
docker compose logs -f                  # 看日志
docker compose down -v                  # 停止并清数据
```

| 模块 | 依赖 | 被依赖 | 耦合度 |

| **services/graph_builder.py** | llm, models | api/graph | 低（独立） |
| **api/documents.py** | document_service, dedup_service | 客户端 | ⬆️ **高**（与两个 Service 紧耦合） |
| **api/questions.py** | anchor_engine | 客户端 | 中 |
| **api/feynman.py** | feynman_evaluator | 客户端 | 低（独立） |
| **api/graph.py** | graph_builder | 客户端 | 低（独立） |

**关键结论：**
- `vectors/store.py` 是共享瓶颈，**必须先做**
- 文档链路（document_service + dedup_service + api/documents）**必须一起做**，耦合最紧
- 费曼 / 图谱 / 卡片+同步 三条链路**互相独立**，可并行开发

## 六、API 与客户端对照清单

## 七、注意事项

1. **每个 Task 完成后必须跑通 `./gradlew assembleDebug`（客户端）和 `pytest`（后端）**

4. **每个 Task 完成后 commit 一次，message 格式：`feat(backend): XXX 模块实现`**
